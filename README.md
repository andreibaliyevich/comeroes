# comeroes
 Heroes of Comics


## Описание проекта

Comeroes — это интернет-магазин комиксов, игрушек, одежды и других товаров, связанных с комиксами и кинокомиксами.

Сайт ориентирован на сеть магазинов в нескольких городах. Для этого в приложении **main** используется модель `Store`, которая хранит контактную информацию и время работы конкретного магазина. В этом же приложении реализована утилита перевода валют по текущему курсу через внешнее API.

Приложение **accounts** отвечает за работу с пользователями. Через модель `OSUser` реализована регистрация и авторизация по email (поле `username` удалено). Добавлены дополнительные поля профиля пользователя.
Регистрация подтверждается переходом по уникальной ссылке из письма, отправленного на email пользователя.

Пользователи имеют возможность:
- добавлять несколько адресов и выбирать основной;
- изменять пароль;
- удалять аккаунт;
- сбрасывать пароль и задавать новый.

Приложение **products** позволяет администраторам добавлять производителей товаров и различные типы товаров с индивидуальными характеристиками.
К товарам можно загружать неограниченное количество изображений (модель `ProductImage`), а также указывать наличие товара в конкретном магазине (модель `Warehouse`).

Пользователи могут оставить один отзыв к одному товару. Отзывы можно лайкать и дизлайкать. При добавлении отзыва рейтинг товара автоматически обновляется через сигнал `post_save`.
Товары можно просматривать в виде сетки или списка. Реализованы сортировка, постраничный вывод и фильтрация товаров с учётом типа товара и валюты, установленной у пользователя.

Приложение **cart** хранит корзину товаров пользователя в сессии. Для управления корзиной используется класс `Cart`. Пользователи могут добавлять и удалять товары из корзины, а также изменять их количество с помощью AJAX-запросов.

Приложение **coupons** позволяет администраторам добавлять купоны с ограниченным сроком действия и процентной скидкой. Пользователь может применить купон на странице корзины.

Приложение **orders** сохраняет информацию о заказах пользователей, включая данные пользователя, адрес доставки или выбранный магазин, применённый купон и скидку, итоговую стоимость, а также список товаров с количеством и ценой.
Зарегистрированные пользователи могут просматривать свои заказы и отслеживать их статус.

Приложение **wishlist** хранит список желаемых товаров пользователей. Добавлять товары в список желаний могут только зарегистрированные пользователи.

Приложение **newsletter** реализует подписку и рассылку email-писем. Подписка подтверждается переходом по уникальной ссылке из письма, отправленного на указанный email. Также учитываются зарегистрированные пользователи с таким же email.

Приложение **support** реализует обмен сообщениями между зарегистрированными пользователями и администраторами для решения возникающих вопросов.

На сайте реализована поддержка русского, белорусского и английского языков.


## Установка

Проект разворачивается с использованием Docker.

### Требования

- Docker
- Docker Compose

### Инструкция по установке

##### 1. Клонировать репозиторий

    git clone https://github.com/andreibaliyevich/comeroes.git

##### 2. Перейти в папку проекта

    cd comeroes

##### 3. Создать файл .env с переменными окружения в папке src

Пример:

    SECRET_KEY=secret_key
    SQL_NAME=sql_name
    SQL_USER=sql_user
    SQL_PASSWORD=sql_password
    EMAIL_HOST_USER=email_host_user
    EMAIL_HOST_PASSWORD=email_host_password
    EMAIL_HOST_RECEIVER=email_host_receiver
    EXCHANGERATE_API_KEY=exchangerate_api_key

##### 4. Создать файл .env с переменными окружения в папке postgres

Пример:

    POSTGRES_DB=sql_name
    POSTGRES_USER=sql_user
    POSTGRES_PASSWORD=sql_password

##### 5. Собрать Docker-образы

    docker compose build

##### 6. Запустить bash-сессию в сервисе django

    docker compose run django bash

##### 7. Применить миграции базы данных

    python manage.py migrate

##### 8. Скомпилировать файлы локализации

    django-admin compilemessages

##### 9. Создать суперпользователя

    python manage.py createsuperuser

##### 10. Выйти из bash-сессии

    exit

##### 11. Запустить все сервисы

    docker compose up
